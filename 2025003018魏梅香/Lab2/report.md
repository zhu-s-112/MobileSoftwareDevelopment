## 二、布局结构简要说明
本次实验基于 **Jetpack Compose** 声明式 UI 框架实现，采用分层组件化设计，核心布局结构如下：
1. **根布局**
   使用 `Column` 组件填充全屏，设置浅紫色背景，控制所有内容**水平居中**，通过 `weight` 权重将屏幕垂直划分为上下区域。
2. **上半部分（权重 0.6f）**
   嵌套 `Column` 组件，内部垂直居中，封装 `CardTop` 组合函数，展示**头像 + 姓名 + 职位**，实现内容在屏幕垂直中点居中。
3. **下半部分（权重 0.4f）**
   嵌套 `Column` 组件，设置**底部对齐 + 水平居中**，封装 `CardBottom` 组合函数，展示联系方式列表，距离底部保留间距。
4. **联系方式行**
   封装 `ContactRow` 组合函数，通过 `Row` 组件横向排列 `Icon` 图标和 `Text` 文字，实现单行联系方式的规整展示。

核心组件：`Column`、`Row`、`Image`、`Text`、`Icon`、`Modifier`

## 三、遇到的问题与解决过程
### 1. 布局位置与效果图不符
- **问题**：上半部分无法精准居中，下半部分位置偏移，排版混乱。
- **解决**：放弃硬编码 `padding`，改用 `weight` 权重分配垂直空间，上半部分搭配 `Arrangement.Center` 居中，下半部分搭配 `Arrangement.Bottom` 靠底，实现精准布局。

### 2. 联系方式无法水平居中
- **问题**：联系方式默认左对齐，整体偏向屏幕一侧。
- **解决**：将下半部分 `Column` 的 `horizontalAlignment` 设置为 `Alignment.CenterHorizontally`，实现**整体居中、行内左对齐**的效果。

### 3. 组件对齐与间距混乱
- **问题**：图标与文字间距不规范，修饰符使用错误导致编译报错。
- **解决**：使用 `Spacer` 控制图标文字间距，统一 `padding` 数值，区分布局对齐参数和组件修饰符的使用场景。

### 4. 界面排版不整洁
- **问题**：联系方式区域无规整间距，视觉效果差。
- **解决**：统一 `ContactRow` 垂直间距，优化文字大小、颜色样式，让界面更专业规整。
