# Lab5：Art Space 应用实验报告

## 一、应用展示内容
本次开发的 Art Space 艺术空间应用以**中国古代名画**为展示主题，共包含 3 幅经典国画作品，分别为：
1. 仙山楼阁图 - 赵伯驹 - 南宋
2. 千里江山图 - 王希孟 - 北宋政和三年
3. 百鸟朝凤图 - 沈铨 - 清乾隆五年

应用支持通过按钮切换作品，图片、作品名称、作者、年代会同步更新，实现完整的数字艺术画廊效果。

## 二、界面结构说明
应用整体使用 Jetpack Compose 构建，界面采用**垂直布局（Column）**作为根容器，清晰划分为三大功能区块：

### 1. 艺术作品展示区
- 使用可组合项：`Box`、`Image`
- 功能：展示画作，带灰色边框与白色背景，模拟画框效果
- 实现：`Box` 作为外层容器提供边框和背景，内部嵌套 `Image` 显示作品图片

### 2. 作品信息展示区
- 使用可组合项：`Column`、`Text`、`Spacer`
- 功能：显示作品标题、作者、创作年代
- 实现：`Column` 垂直排列标题与作者信息，`Spacer` 控制文本间距

### 3. 切换控制按钮区
- 使用可组合项：`Row`、`Button`、`Spacer`
- 功能：提供“上一个”“下一个”按钮，用于切换作品
- 实现：`Row` 水平排列两个按钮，中间添加间距保证美观

整体嵌套结构：
`Column` → 作品展示区 → 作品信息区 → 控制按钮区
布局对齐方式合理，界面整洁、层次分明。

## 三、Compose 状态管理实现
### 1. 定义状态变量
```kotlin
var currentArtIndex by remember { mutableStateOf(1) }
```

### 2. 状态作用

`currentArtIndex` 用于记录当前显示的作品编号（1、2、3）。图片资源、标题、作者、年代均通过该状态变量使用 `when` 表达式动态获取。

### 3. 状态驱动界面

当状态变量发生变化时，Compose 自动重组界面组件，更新所有展示内容，实现无刷新、自动更新的交互效果。

## 四、Next / Previous 按钮逻辑说明

### 1. 上一个按钮（Previous）逻辑

- 判断当前作品是否为第 1 幅
- 如果是第 1 幅，则切换到第 3 幅（最后一幅）
- 否则，作品编号减 1，切换到上一幅
- 实现循环切换效果

### 2. 下一个按钮（Next）逻辑

- 判断当前作品是否为第 3 幅
- 如果是第 3 幅，则切换到第 1 幅（第一幅）
- 否则，作品编号加 1，切换到下一幅
- 实现循环切换效果

按钮边界处理完整，不会出现越界或崩溃情况。

## 五、遇到的问题与解决过程

### 问题 1：画框边框显示异常，被背景遮挡

- **原因：** 最初使用 `Surface` 嵌套导致层级混乱
- **解决：** 改用 `Box` 作为外层容器，先设置边框再设置背景，效果正常

### 问题 2：按钮点击后界面不更新、作品不切换

- **原因：** 状态变量未使用 `by` 委托，无法触发重组
- **解决：** 使用 `by remember { mutableStateOf(1) }` 正确声明状态

### 问题 3：界面元素拥挤、布局不美观

- **解决：** 使用 `Spacer` 和 `Modifier.padding` 增加间距，调整对齐方式，优化界面排版

### 问题 4：项目 Gradle 同步失败，无法加载

- **原因：** `build.gradle` 中 `compileSdk` 语法错误
- **解决：** 修改为标准写法 `compileSdk = 36`，重新同步后项目正常运行
