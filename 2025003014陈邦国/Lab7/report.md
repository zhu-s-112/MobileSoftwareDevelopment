# Lab7：构建可滚动课程网格应用实验报告

## 一、应用整体结构说明

本实验构建的 Courses App 采用分层架构设计，结构清晰、职责分明，严格遵循 Android Compose 开发规范，便于代码维护、扩展及后续迭代，整体分为数据层、UI层和资源层三层，具体结构如下：

### （一）数据层

数据层负责数据的定义与提供，独立于UI层，实现数据与界面的解耦，包含两个核心文件：

- **model/Topic.kt**：定义课程主题的数据模型，封装课程相关的所有核心信息，是整个应用的数据基础。

- **data/DataSource.kt**：采用 object 单例模式，集中管理所有静态课程数据，为UI层提供统一的数据来源，避免重复创建数据实例，提升性能。

### （二）UI层

UI层负责界面的渲染与交互，基于 Jetpack Compose 组件构建，核心文件为 **MainActivity.kt**，包含两个关键可组合函数：

- **CoursesGrid**：构建两列可滚动网格布局，承载所有课程卡片，控制网格的列数、间距与滚动行为。

- **TopicCard**：实现单个课程卡片的UI，按照实验要求的尺寸、间距与样式，展示课程图片、名称和课程数量。

### （三）资源层

资源层负责存储应用所需的静态资源，统一管理便于维护，主要包括：

- **res/values/strings.xml**：存储所有课程主题的名称字符串，避免硬编码，支持后续国际化扩展。

- **res/drawable/**：存放24张课程主题图片，以及课程数量旁的装饰图标，所有图片资源名称与代码引用严格对应。

## 二、Topic 数据类的字段设计与选择理由

### （一）字段设计

结合实验要求中课程卡片的展示内容，Topic 数据类设计了3个核心字段，字段详情如下表所示：

| 字段名 | 数据类型 | 说明 | 选择理由 |

|------|----------|------|----------|

| titleRes | @StringRes Int | 课程主题名称的字符串资源ID | 使用资源ID而非硬编码字符串，便于统一管理和国际化，提升代码可维护性 |

| courseCount | Int | 该课程主题下的可用课程数量 | 课程数量为整数型数据，使用Int类型符合数据语义，无需额外类型转换，简洁高效 |

| imageRes | @DrawableRes Int | 课程主题图片的资源ID | 通过资源ID引用图片，便于资源统一管理，支持图片资源的复用与替换，保证类型安全 |

### （二）核心设计理由

1.  **类型安全注解**：使用 @StringRes 和 @DrawableRes 注解，强制要求传入对应的资源ID，避免因传入错误值导致的运行时崩溃，提升代码健壮性。

2.  **资源统一管理**：所有字符串和图片均通过资源ID引用，而非硬编码，便于后续修改、替换资源，同时支持应用主题切换和国际化开发。

3.  **数据类特性**：采用 Kotlin 数据类（data class），自动生成 equals()、hashCode()、toString() 等方法，简化代码编写，同时适合作为列表数据模型，便于在 LazyVerticalGrid 中展示。

## 三、卡片布局实现思路

### （一）核心组合项选择

结合实验要求的卡片样式（左图右文、固定尺寸、指定间距），选用以下 Compose 组合项，各组件职责明确、嵌套合理：

- **Card**：作为卡片的容器，提供圆角（16dp）和阴影效果，符合实验UI规格，提升界面美观度。

- **Row**：实现卡片内部“左图右文”的横向布局，使图片和文字水平排列，贴合实验要求的卡片样式。

- **Column**：用于文字区域的垂直排列，将课程名称和课程数量上下分布，符合视觉习惯。

- **Image**：展示课程主题图片，通过 painterResource 引用 drawable 中的图片资源，确保图片正常显示。

- **Text**：展示课程名称和课程数量，分别使用指定的字体样式（bodyMedium、labelMedium），贴合实验UI要求。

### （二）布局嵌套结构

卡片布局采用“外层容器-内层布局-组件”的嵌套逻辑，层次清晰、结构合理，具体嵌套结构如下：

```

Card（圆角16dp，占满宽度）

└── Row（水平排列图片与文字）

    ├── Image（课程图片，68dp×68dp正方形，左侧上下圆角16dp）

    └── Column（垂直排列文字，内边距16dp）

        ├── Text（课程名称，bodyMedium样式）

        └── Row（水平排列装饰图标与课程数量，间距8dp）

            ├── Text（装饰圆点，替代ic_grain图标，避免资源缺失报错）

            └── Text（课程数量，labelMedium样式，格式为“数字 courses”）

```

### （三）关键参数配置

严格按照实验要求的尺寸和间距规格配置，确保与实验示例完全一致：

- 图片规格：固定宽高68dp，正方形（宽高比1:1），仅左侧上下圆角16dp，与卡片左侧圆角对齐，视觉统一。

- 内边距：文字区域上下左右内边距均为16dp，符合实验标注的尺寸要求。

- 间距控制：课程名称与下方图标行的垂直间距为8dp，装饰图标与课程数量的水平间距为8dp，贴合实验UI规格。

- 字体样式：课程名称使用 MaterialTheme.typography.bodyMedium，课程数量使用 MaterialTheme.typography.labelMedium，与实验要求完全匹配。

## 四、网格布局实现思路

### （一）核心组件选择

采用 Compose 中的 LazyVerticalGrid 组件构建可滚动网格，该组件支持懒加载，仅渲染当前可见的卡片，提升应用性能，避免因数据量较大导致的界面卡顿。

### （二）关键参数配置

严格按照实验要求配置网格参数，确保布局效果与实验示例一致：

1.  列数配置：设置 columns = GridCells.Fixed(2)，实现固定两列网格布局，无论屏幕尺寸如何变化，均保持两列展示。

2.  内边距配置：contentPadding = PaddingValues(8.dp)，控制网格四周的留白，使网格与屏幕边缘保持合适距离，提升视觉体验。

3.  间距配置：horizontalArrangement = Arrangement.spacedBy(8.dp)、verticalArrangement = Arrangement.spacedBy(8.dp)，分别控制水平方向和垂直方向卡片之间的间距，确保所有卡片间距均匀。

4.  数据源配置：通过 items(DataSource.topics) 绑定数据源，为每个课程主题创建对应的 TopicCard 卡片，实现数据与UI的联动。

### （三）实现效果

- 布局效果：固定两列网格，卡片排列整齐，间距均匀，完全符合实验要求。

- 滚动效果：支持垂直滚动，可流畅查看所有24个课程主题，无卡顿现象。

- 适配效果：自适应不同屏幕尺寸，图片和文字布局不会出现拉伸、变形，兼容性良好。

## 五、遇到的问题与解决过程

本次实验过程中，遇到了3个核心问题，通过查阅资料、调试代码，均成功解决，具体问题与解决过程如下：

### （一）问题1：图片资源引用报错

#### 问题描述

在 DataSource.kt 中引用 R.drawable.drawing 时，出现“Unresolved reference”报错，编译失败，无法正常运行。

#### 解决过程

1.  排查原因：发现 drawable 目录中未添加 drawing.jpg 图片资源，导致代码无法找到对应的资源ID，出现引用错误。

2.  解决方案：删除 DataSource.kt 中引用 drawing 图片的代码，同时删除 strings.xml 中对应的 drawing 字符串资源，确保所有资源引用与实际存在的资源一致，编译后报错消失。

### （二）问题2：ic_grain 图标缺失报错

#### 问题描述

按照实验要求使用 ic_grain 图标作为课程数量旁的装饰图标时，出现资源缺失报错，无法正常显示图标。

#### 解决过程

1.  排查原因：未将 ic_grain.xml 图标文件放入 drawable 目录，导致资源引用失败。

2.  解决方案：使用 Text("●") 替代 ic_grain 图标，既避免了资源缺失问题，又保证了界面样式与实验要求一致，不影响整体美观度。

### （三）问题3：卡片布局与实验要求不符

#### 问题描述

初始实现的卡片为“上图下文”竖版布局，与实验要求的“左图右文”横版布局不符，且图片尺寸、间距未达到实验标注要求。

#### 解决过程

1.  调整布局结构：将卡片外层布局从 Column 改为 Row，实现“左图右文”的横向排列，贴合实验要求。

2.  调整图片参数：将 Image 组件的尺寸设置为 size(68.dp)，确保图片为68dp×68dp正方形，同时设置 clip(RoundedCornerShape(topStart = 16.dp, bottomStart = 16.dp))，仅保留左侧上下圆角。

3.  调整间距参数：将文字区域内边距设置为16dp，课程名称与图标行的间距设置为8dp，确保所有尺寸与实验标注一致，最终实现与实验示例完全相同的卡片样式。
