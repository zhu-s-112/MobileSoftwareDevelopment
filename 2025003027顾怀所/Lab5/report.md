# Lab5：创建 Art Space 应用

# 一、应用展示内容

## 本次开发的Art Space应用，以法国印象派大师克劳德·莫奈的三幅经典代表作为展示主题，共包含3幅艺术作品，实现作品循环切换展示功能，具体作品如下：

1. 《安提比斯的松树》：创作于1888年，是莫奈在地中海沿岸安提比斯创作的风景作品，以明快的色彩勾勒出阳光下的松树与海岸景致，尽显印象派光影捕捉的精髓。

2. 《日出·印象》：创作于1872年，作为印象主义绘画的开山之作，以朦胧的笔触、柔和的光影，描绘了勒阿弗尔港口日出时分的朦胧美景，奠定了印象派的艺术风格。

3. 《吉维尼花园的鸢尾花》：创作于1900年，是莫奈晚年定居吉维尼花园时创作的花卉作品，用浓烈且层次丰富的色彩，展现花园中鸢尾花的蓬勃生机，是其后期花卉题材的经典之作。

应用通过简洁优雅的界面，展示艺术作品画作、作品标题、创作者及创作年份，搭配暗紫色系交互按钮，整体风格贴合艺术展示场景，视觉效果和谐统一。

# 二、界面结构说明

## 应用整体采用Jetpack Compose声明式UI构建，整体布局以Column垂直布局为核心，自上而下划分为三大逻辑区块，各区块层级清晰、布局合理，具体结构如下：

（一）艺术作品展示区

- 核心组件：Box+Image

- 实现方式：通过Box容器打造画框效果，添加阴影、白色背景、浅灰色边框，模拟真实画作相框；内部嵌套Image组件加载艺术作品图片，设置固定高度与内容缩放模式，保证画作展示完整且美观。

- 布局效果：占据界面主要视觉空间，居中展示作品，边缘留有合适内边距，提升视觉质感。

（二）艺术作品信息区

- 核心组件：Column+Text

- 实现方式：通过Column垂直排列两组文本，上方为加粗大号字体的作品标题，突出核心信息；下方为灰色常规字体，同步展示艺术家姓名与创作年份，信息层级分明，便于用户查看。

- 布局效果：全屏宽度居中展示，与上下区块保留合适间距，文本对齐整齐，阅读体验舒适。

（三）作品切换控制区

- 核心组件：Row+Button

- 实现方式：通过Row水平布局，均匀排列Previous（上一个）和Next（下一个）两个按钮，按钮采用统一的暗紫色系（色值#4A2C6B），搭配白色按钮文字，尺寸固定，交互区域清晰。

- 布局效果：位于界面底部，与上方信息区保留间距，按钮对称分布，点击区域充足，操作便捷。

整体布局通过Modifier修饰器设置内边距、对齐方式、尺寸大小，结合Spacer组件控制元素间距，适配不同屏幕尺寸，界面布局规整、美观大方。

# 三、Compose状态管理实现

## 应用采用Jetpack Compose的状态管理机制，实现作品切换的动态交互，核心实现逻辑如下：

1. 状态变量定义

定义可观察的整型状态变量currentArtwork，用于记录当前展示的作品索引，初始值为1，通过remember+mutableIntStateOf实现状态存储与界面重组监听：

var currentArtwork by remember { mutableIntStateOf(1) }

2. 状态联动界面

通过when多分支表达式，根据currentArtwork的取值，动态匹配对应的作品图片、标题、艺术家、年份信息，实现界面内容随状态值实时更新。

3. 状态更新机制

用户点击切换按钮时，修改currentArtwork的取值，Compose会自动监听状态变化，触发界面重组，刷新图片与文本信息，无需手动更新UI组件。

该状态管理方式简洁高效，通过单一状态变量控制所有动态界面元素，符合Compose声明式编程思想，实现状态与界面的单向绑定。

# 四、按钮交互逻辑说明

## 应用通过Previous和Next按钮实现作品循环切换，核心逻辑采用when多分支表达式实现，避免繁琐的if-else嵌套，逻辑清晰易懂：

（一）Next（下一个）按钮逻辑

- 核心逻辑：点击后作品索引自增，若当前为最后一幅作品（索引3），则切换至第一幅作品（索引1），实现循环切换；

- 代码实现：

onNext = {
    currentArtwork = when (currentArtwork) {
        3 -> 1
        else -> currentArtwork + 1
    }
}

（二）Previous（上一个）按钮逻辑

- 核心逻辑：点击后作品索引自减，若当前为第一幅作品（索引1），则切换至最后一幅作品（索引3），实现循环切换；

- 代码实现：

onPrevious = {
    currentArtwork = when (currentArtwork) {
        1 -> 3
        else -> currentArtwork - 1
    }
}

两个按钮的交互逻辑互为补充，覆盖所有边界情况，实现作品无缝循环切换，交互流程流畅无卡顿。

# 五、实验遇到的问题与解决过程

（一）问题1：按钮颜色与艺术应用风格不匹配

- 问题描述：初始按钮采用蓝色系，与艺术展示的优雅风格违和，视觉效果不佳。

- 解决方法：将按钮背景色修改为暗紫色系（#4A2C6B），该色调沉稳高级，贴合艺术应用的视觉定位，通过ButtonDefaults.buttonColors统一设置两个按钮的容器颜色，保证风格统一。

（二）问题2：作品图片展示变形、无边框质感

- 问题描述：图片直接展示无修饰，缺乏画框效果，且缩放模式不当导致画面拉伸变形。

- 解决方法：使用Box容器嵌套图片，添加阴影、边框、白色背景模拟画框；设置ContentScale.Crop缩放模式，保证图片按比例填充展示区域，避免画面变形。

（三）问题3：状态更新后界面无响应

- 问题描述：初始使用普通变量存储作品索引，点击按钮后变量值改变，但界面未刷新。

- 解决方法：改用Compose推荐的remember+mutableIntStateOf定义可观察状态变量，让Compose能够监听状态变化，自动触发界面重组，实现界面实时更新。

（四）问题4：界面元素间距混乱、对齐不规整

- 问题描述：各区块元素间距不一，文本、按钮对齐混乱，整体界面杂乱。

- 解决方法：通过Modifier的padding、align、horizontalAlignment等属性，统一设置对齐方式与间距；利用Row的Arrangement.SpaceEvenly实现按钮均匀分布，优化界面布局规整度。
