## 二、布局结构说明
整体采用 Jetpack Compose 声明式布局，核心结构如下：

### 1. 整体容器
使用 Box 作为最外层容器，设置月白星紫渐变背景（Color(0xFFF8F8FF) 到 Color(0xFF9370DB)）。
内容区域使用 Column 垂直排列，分为上半部分（个人信息）和下半部分（联系方式）。
通过 padding(top = 100.dp, bottom = 150.dp) 控制上下边距，确保所有内容完整显示在屏幕内。

### 2. 上半部分（个人信息）
组件：CardTop
布局：Column 垂直居中排列
元素：
Image：圆形头像，无背景色干扰
Text：姓名（32.sp 加粗白色）、职位（20.sp 月白色）
间距：使用 Arrangement.spacedBy(20.dp) 控制元素间呼吸感

### 3. 下半部分（联系方式）
组件：CardBottom + ContactRow
布局：Column 包裹多行 ContactRow，每行使用 Row 水平排列图标与文字
元素：
Icon：使用 Material Icons（电话、邮箱、分享），星紫色（#9370DB）
Text：联系方式信息（18.sp 白色）
封装：将「图标+文字」封装为 ContactRow 组件，提高代码复用性

## 三、遇到的问题与解决过程

### 问题1：头像后紫色圆圈背景影响美观
问题描述：初始代码中头像带有月白到星紫的渐变圆形背景，与整体背景叠加后显得杂乱。
解决方法：
移除头像 Modifier 中的 background(brush = ...) 和多余形状修饰；
保留 size(120.dp) 控制头像大小，使用 .clip(CircleShape) 实现无背景纯圆形头像；
让头像与渐变背景自然融合，提升整体美观度。

### 问题2：页面内容布局拥挤、边缘贴边
问题描述：初始布局上下边距过小，元素间距紧凑，视觉压抑。
解决方法：
调整整体 Column 的上下内边距为 top = 100.dp, bottom = 150.dp；
调整分区间距为 spacedBy(40.dp)，联系方式行间距为 spacedBy(24.dp)；
优化文字字号与权重，调整水平内边距，布局更加协调舒适。

### 问题3：文字与背景对比度不足
问题描述：浅色文字在渐变背景下可读性较差。
解决方法：
姓名使用纯白色，职位使用月白色，保证阅读清晰；
图标统一使用星紫色配色，和主题风格统一、形成视觉对比；
规范基础字号，避免文字过小看不清。

### 问题4：联系方式内容被截断，邮箱和社交账号不显示
问题描述：仅电话正常展示，邮箱、社交账号被挤出屏幕底部。
原因分析：
CardBottom 额外设置顶部边距造成整体上移；原有底部留白不足，容纳不下三行内容。
解决方法：
移除 CardBottom 多余顶部 padding；
加大全局底部边距至150.dp；
优化分区垂直间距，修复后所有联系方式完整正常显示。
